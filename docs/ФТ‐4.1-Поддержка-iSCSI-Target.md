## Описание
Система хранения данных (СХД) должна поддерживать одновременную работу множества независимых iSCSI Target’ов, каждый из которых предоставляет доступ к выделенным LUN’ам по протоколу iSCSI. Каждый Target должен иметь уникальную конфигурацию: IQN, сетевые интерфейсы, политики доступа, параметры аутентификации. Поддержка должна обеспечивать масштабируемость, безопасность и изоляцию между Target’ами.
## Функциональные характеристики
1. Поддержка множественных iSCSI Target’ов
* •	 СХД поддерживает не менее 1000 независимых iSCSI Target’ов на кластер
* •	Каждый Target:
* •	Имеет уникальный IQN (iSCSI Qualified Name).
* •	Может быть привязан к одному или нескольким IP-адресам.
* •	Управляет собственным набором LUN и политиками доступа.
* •	Поддержка активно-активной архитектуры: Target’ы могут быть распределены между контроллерами для балансировки нагрузки и отказоустойчивости.
2. Управление IQN и идентификацией
* •	Автоматическая генерация [IQN по правилу]("D:\Users\A N D R U\MyTraining\GitHub\API\CXD\docs\Rule_01_IQN.md") или задаваться вручную (`TBD`).
* •	Запрет дублирования IQN в пределах СХД.
* •	Поддержка зарезервированных префиксов для мультитенантных сценариев.
3. Управление LUN и привязка к Target’ам
* •	Один LUN может быть привязан к одному или нескольким Target’ам (при соблюдении политик безопасности).
* •	Поддержка:
* •	[Thin-provisioning](https://github.com/sunname91-git/API/wiki/_new#scroll-bookmark-45)
* •	[Thick-provisioning](https://github.com/sunname91-git/API/wiki/_new#scroll-bookmark-38)
* •	Максимум 512 LUN на один Target
4. Контроль доступа и безопасность
* •	Поддержка ACL (Access Control Lists) на уровне каждого Target:
* •	Разрешение/запрет подключения по IQN Initiator’а (группы Initiator’ов).
* •	Аутентификация:
* •	[CHAP](https://confluence.is.open-yard.com/pages/viewpage.action?pageId=87622881&src=contextnavpagetreemode) (односторонняя и двусторонняя Mutual CHAP).
* •	Уникальные CHAP-учётные записи на Target.
* •	Изоляция политик: настройки одного Target не влияют на другие.
5. Мониторинг и диагностика
5.1 Для каждого Target доступна статистика:
* •	Количество активных сессий.
* •	Объём переданных данных (чтение/запись).
* •	IOPS, пропускная способность, задержка.
* •	Состояние подключения (включен/отключен).
5.2 Ведение логов:
* •	Подключения/отключения Initiator’ов.
* •	Ошибки аутентификации.
* •	События изменения конфигурации.
