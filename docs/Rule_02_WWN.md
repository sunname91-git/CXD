WWN (World Wide Name) — это уникальный идентификатор для устройств Fibre Channel (FC). Формирование WWN зависит от типа устройства и производителя.
В [RFC4122](https://www.rfc-editor.org/rfc/rfc4122) (Universally Unique IDentifier, UUID) и других связанных стандартах описываются требования к формату WWN (World Wide Name), который используется для уникальной идентификации устройств в сетях Fibre Channel (FC). WWN является ключевым элементом в FC-сетях, так как он обеспечивает глобально уникальную идентификацию устройств, таких как хост-адаптеры, коммутаторы и дисковые массивы.
## Общие требования к формату WWN
1. Длина WWN :
* •	WWN представляет собой 64-битное (8 байт) или 128-битное (16 байт) значение, в зависимости от типа.
* •	Наиболее распространённый формат — 64-битный WWN.
2. Структура WWN :
* •	WWN состоит из двух основных частей: **NAA (Network Address Authority)** и **Vendor-Specific Identifier** .
* •	Первые несколько битов определяют тип WWN (формат NAA).
* •	Остальная часть содержит уникальный идентификатор, назначенный производителем устройства.
3. Типы WWN (NAA) :
* •	**NAA = 1** : Локально назначенный адрес (не рекомендуется для глобального использования).
* •	**NAA = 2** : IEEE расширенный адрес (использует MAC-адрес производителя).
* •	**NAA = 5** : IEEE зарегистрированный адрес (наиболее распространённый формат, использует OUI — Organizationally Unique Identifier).
* •	**NAA = 6** : Расширенный IEEE зарегистрированный адрес (128-битный формат).
4. Формат записи :
* •	WWN обычно записывается в шестнадцатеричном формате, разделённом двоеточиями или без разделителей.
* •	Пример: 50:06:0e:80:05:a8:c1:00 или 50060e8005a8c100.
5. Уникальность :
* •	WWN должен быть глобально уникальным. Это достигается за счёт использования OUI (Organizationally Unique Identifier), который присваивается IEEE каждому производителю, и уникального Vendor-Specific Identifier.
6. Соответствие стандартам :
* •	WWN должен соответствовать стандарту IEEE 802 (MAC-адреса) и спецификациям Fibre Channel (например, FC-PH, FC-GS).
## Требования к адаптерам FC
Для адаптеров Fibre Channel (например, HBA — Host Bus Adapter) WWN должен:
1. Быть глобально уникальным.
2. Использовать формат, совместимый с Fibre Channel и IEEE.
3. Поддерживать как минимум один из типов NAA (чаще всего NAA = 5).
4. Предоставляться в документации или интерфейсах управления адаптера (например, через CLI или GUI).
## Примеры WWN
* •	**64-битный WWN (NAA = 5)** : 50:06:0e:80:05:a8:c1:00
* •	**128-битный WWN (NAA = 6)** : 60:06:0e:80:05:a8:c1:00:12:34:56:78:9a:bc:de:f0
