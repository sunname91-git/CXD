## Описание
Система должна обеспечивать механизм ограничения доступа к логическим устройствам хранения (LUN) на уровне логических [хостов iSCSI](https://github.com/sunname91-git/API/wiki/_new#scroll-bookmark-111). Доступ к LUN должен предоставляться только тем iSCSI-инициаторам, которые входят в состав хоста, явно сопоставленного с данным LUN.
Доступ к LUN разрешается не напрямую индивидуальным инициаторам, а через членство в хосте, которому предоставлено сопоставление (mapping) с LUN.
## Требуемая функциональность
1. Сопоставление LUN с хостом (Host-to-LUN Mapping)
Администратор должен иметь возможность сопоставить LUN с конкретным логическим хостом (группой инициаторов). Доступ к LUN получают все iSCSI-инициаторы, входящие в состав этого хоста.
2. Ограничение доступа по умолчаниюВсе LUN по умолчанию должны быть недоступны ни одному хосту. Доступ разрешается исключительно через явное сопоставление LUN с хостом.
3. Поддержка множественных хостов на один LUN
Один LUN может быть сопоставлен с несколькими хостами, если это необходимо для кластерных или общих сценариев использования (shared access).
4. Удаление и изменение сопоставленийАдминистратор может в любой момент отменить сопоставление LUN с хостом. После этого все инициаторы, входящие в этот хост, теряют доступ к LUN при повторном подключении или пересканировании.
5. Проверка подлинности и авторизацииПри подключении iSCSI-инициатора система должна:
•	Проверить его IQN на наличие в составе зарегистрированного хоста.
•	Проверить, сопоставлен ли этот хост с запрашиваемым LUN.
•	Host Mapping должен корректно работать в связке с аутентификацией [CHAP](https://confluence.is.open-yard.com/pages/viewpage.action?pageId=87622881&src=contextnavpagetreemode)
•	Разрешить доступ только при положительных результатах всех проверок.
6. Логирование
Все операции по сопоставлению LUN с хостами должны фиксироваться в журнале действий пользователя с указанием:
•	даты и времени (с точностью до секунды, с указанием часового пояса или в формате UTC),
•	идентификатора пользователя (администратора), выполнившего операцию,
•	наименования и идентификатора Volume/LUN,
•	имени хоста,
•	типа операции (создание, изменение, удаление),
•	IP-адрес или идентификатор источника запроса (например, с какого административного интерфейса выполнено действие — GUI, CLI).

## Критерии приемки
1. Доступ к LUN имеют только те iSCSI-инициаторы, которые:
•	Входят в состав зарегистрированного логического хоста,
•	И этот хост явно сопоставлен с данным LUN.
