## Описание
Система хранения данных (СХД) должна предоставлять возможность управления хостами iSCSI, где хост представляет собой логическую сущность, объединяющую один или несколько iSCSI-инициаторов для централизованного управления доступом к LUN'ам и настройками подключения.

## Требуемая функциональность
1. Создание хоста iSCSI:
•	Администратор должен иметь возможность создавать новый хост iSCSI через графический интерфейс (GUI) или CLI.
•	При создании хоста требуется указать:
- [Имя ]("D:\Users\A N D R U\MyTraining\GitHub\API\CXD\docs\Rule_07_Наименование-сущностей-(Name).md")хоста (уникальное в пределах СХД).
- [Описание ]("D:\Users\A N D R U\MyTraining\GitHub\API\CXD\docs\Rule_08_Описание-(Description).md")хоста (опционально).
- Список iSCSI-инициаторов (IQN), которые будут ассоциированы с хостом.
2. Добавление iSCSI-инициаторов:
•	Поддерживается ручное добавление iSCSI-инициаторов по их [IQN ](Rule_01_IQN.md)(iSCSI Qualified Name).
•	Система не должна позволять вручную добавлять инициаторы, которые уже активны в сети и автоматически обнаружены системой.
•	Для инициаторов, обнаруженных автоматически (через функцию iSCSI Discovery), система должна предлагать опцию их привязки к существующему или новому хосту, но только после подтверждения администратором.
•	Каждый инициатор может быть привязан только к одному хосту.
3. Редактирование хоста:
•	Администратор может:
- Изменять имя и описание хоста.
- Добавлять новых инициаторов (вручную, если они не обнаружены системой).
- Удалять инициаторов из состава хоста.
•	Запрещено редактирование хоста, если он используется в активных мэппингах LUN.
4. Удаление хоста:
•	Хост может быть удалён только если:
- К нему не привязаны LUN'ы.
- Он не участвует в активных сессиях iSCSI.
•	При удалении хоста все связанные с ним ручные инициаторы удаляются из системы.
5. Просмотр и мониторинг:
•	Система должна отображать:
- Список всех хостов iSCSI.
- Состояние каждого хоста (активен/неактивен).
- Список инициаторов, ассоциированных с хостом.
- Информацию о подключённых сессиях (при наличии).
•	Для каждого инициатора указывается источник добавления: "вручную" или "автообнаружение".
6. Автоматическое обнаружение инициаторов:
•	Система должна периодически сканировать сеть на наличие активных iSCSI-инициаторов (Discovery по iSNS или мониторинг трафика iSCSI-портов).
•	Обнаруженные инициаторы отображаются в интерфейсе в отдельной секции ("Обнаруженные инициаторы").
•	Администратор может вручную назначить обнаруженный инициатор существующему хосту или создать для него новый хост.
